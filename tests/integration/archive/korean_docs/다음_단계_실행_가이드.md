## 📋 즉시 실행 가능한 작업

### 방법 1: HYSPLIT Web 수동 실행 (권장) ⭐

**소요 시간:** 30-60분  
**난이도:** 쉬움  
**정확도:** 가장 높음

#### 단계별 실행

**1단계: 도우미 스크립트 실행**

```bash
python tests\integration\hysplit_web_helper.py
```

이 스크립트는 다음을 제공합니다:
- 8개 지역의 정확한 좌표
- 복사-붙여넣기 가능한 설정 값
- 단계별 체크리스트
- 다운로드 진행 상황 확인

**2단계: HYSPLIT Web 접속**

웹사이트: https://www.ready.noaa.gov/HYSPLIT_traj.php

**3단계: 공통 설정 입력 (모든 지역 동일)**

```
Meteorology:
  Model: GFS (0.25 degree)
  Start Time: 2026-02-14 00:00 UTC

Trajectory:
  Direction: Backward
  Duration: 24 hours
  Vertical Motion: Model Vertical Velocity

Output:
  Interval: 1 hour
```

**4단계: 각 지역 실행 (8개)**

| 순서 | 지역 | 위도 | 경도 | 고도 |
|------|------|------|------|------|
| 1 | 서울 | 37.5 | 127.0 | 850 |
| 2 | 부산 | 35.1 | 129.0 | 850 |
| 3 | 제주 | 33.5 | 126.5 | 850 |
| 4 | 도쿄 | 35.7 | 139.7 | 850 |
| 5 | 오사카 | 34.7 | 135.5 | 850 |
| 6 | 베이징 | 39.9 | 116.4 | 850 |
| 7 | 상하이 | 31.2 | 121.5 | 850 |
| 8 | 타이베이 | 25.0 | 121.5 | 850 |

각 지역마다:
1. 위도/경도/고도 입력
2. Run 클릭
3. 계산 완료 대기 (1-2분)
4. "Trajectory Endpoints" 다운로드
5. 파일명을 `tdump_<지역명>.txt`로 변경
6. `tests\integration\hysplit_web_data\` 폴더에 저장

**5단계: 비교 실행**

```bash
python tests\integration\multi_location_24h_comparison.py --compare
```

목표: 99%+ 일치

## ❓ 자주 묻는 질문

### Q1: HYSPLIT Web 접속이 안 됩니다.
A: NOAA 서버 점검 시간일 수 있습니다. 다른 시간대에 재시도하세요.

### Q2: 계산이 실패합니다.
A: 시작 위치가 GFS 데이터 범위 내인지 확인하세요 (110-150°E, 20-50°N).

### Q3: tdump 파일 형식이 맞지 않습니다.
A: "Trajectory Endpoints"를 다운로드했는지 확인하세요 (다른 형식 아님).

### Q4: 비교 결과가 이상합니다.
A: PyHYSPLIT과 HYSPLIT Web의 설정이 동일한지 확인하세요.

### Q5: 시간이 너무 오래 걸립니다.
A: 각 지역당 3-5분이 정상입니다. 총 30-60분 소요됩니다.
